Requirements

Before you can start heading over to the components sections, go through the list of requirements and fulfill them if needed.

Successful Setup

A successful Setup of the buildserver is a strong requirement, since it includes setting up the repositories that are used by users to supply the build instructions to the buildserver.

At the end of the setup, the Administrator is instructed to create the User-Documentation. See the (example setup @ HTWG)

Hardware Deployment

To deploy a complete system, successful of all components need to be available. This means, single components can be built and deployed if there exists previous successful builds for the target platform. After the required build have successfully completed you can finally deploy to hardware using the flashtool

#!/bin/bash -ex
CHROOT_PATH=$(pwd)
sed -i '1d' ${CHROOT_PATH}/etc/shadow
echo root:$(python -c 'import crypt; print crypt.crypt("toor","$6$somesalt$")'):10770:0::::: >> ${CHROOT_PATH}/etc/shadow
